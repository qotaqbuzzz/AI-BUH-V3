# SKILL: Подготовка НДС 300.00 (VAT Return Prep)

**Trigger:** "НДС 300" / "декларация НДС" / "подготовить 300.00" / "VAT return"
**Role:** Налоговый бухгалтер, готовящий квартальную 300.00 по НК РК.

## Required inputs
```
quarter: Q1(01-03) / Q2(04-06) / Q3(07-09) / Q4(10-12)
year, tenantId, organizationGuid
dateFrom = YYYY-QM1-01, dateTo = YYYY-QM3-last
```

## НДС rules (never break these)
```
Начисленный НДС = база реализации × 12%   ← NOT × 12/112 (common error)
К возмещению ≤ закупки × 12/112           ← here 12/112 is correct (extract from gross)
Deadline: 300.00 by 15th of month after quarter end
ЭСФ: every Реализация must have matching record in InformationRegister_АктуальныеЭСФ
```

## Workflow

**Step 1 — НДС начисленный (output tax)**
```
onec_validate_vat_charged_vs_revenue(dateFrom, dateTo, organizationGuid)
  → delta > 1% → FAIL → Step 1a
  → delta ≤ 1% → OK
```
```
Step 1a: onec_drill_vat_documents(dateFrom, dateTo, organizationGuid)
  → shows per-document deviation
  → identify docs with missing 3131 posting
```

**Step 2 — ЭСФ coverage**
```
onec_validate_esf_coverage(dateFrom, dateTo, organizationGuid)
  → missingCount > 0 → CRITICAL
  Step 2a: onec_drill_missing_esf(dateFrom, dateTo, organizationGuid)
  → list every Реализация GUID without ЭСФ
  → user must register in ИС ЭСФ (kz.kgd.gov.kz) and link in 1С
```

**Step 3 — НДС к возмещению (input tax)**
```
onec_validate_vat_recoverable_vs_purchases(dateFrom, dateTo, organizationGuid)
  → excess detected → show delta, ask user to verify supplier ЭСФ validity
```

**Step 4 — Net НДС position**
```
onec_get_osv(dateFrom, dateTo, organizationGuid)
  → extract: 3131 Кт turnover (charged)
             3130 Кт closing (payable)
             1421 Дт closing (recoverable)
  → net = 3131 - 1421
  → net > 0 → К уплате: [amount] ₸ by [deadline]
  → net < 0 → К возмещению: [amount] ₸ (зачёт или возврат)
```

**Step 5 — ESF status spot-check**
```
onec_get_esf_status(organizationGuid, dateFrom, dateTo)
  → count, registered date range
  → if count < sales document count → mismatch, cross-check
```

## Output format
```
## НДС за [Q] [YYYY]
Начисленный НДС (3131):   X ₸    Статус: ✅/❌
ЭСФ покрытие:             N/N    Статус: ✅/❌ (missing: M)
НДС к возмещению (1421):  X ₸    Статус: ✅/❌
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Нетто к уплате/возмещению: ±X ₸
Срок уплаты: [date]
Проблемы: [list or "нет"]
```
